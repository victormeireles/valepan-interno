# Painel embalagem lê Supabase (Fase B.3) — Plano de implementação

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrar leitura do painel embalagem para Supabase com UI hierárquica pedido→lotes, botão Novo lote, e meta editando pedido canônico.

**Architecture:** `PainelEmbalagemService` monta `pedidos[]` com `lotes[]` aninhados; GET painel só DB; realizado/meta refatorados; novas rotas pedido CRUD e resolver-linha para escrita legada na planilha.

**Tech Stack:** Next.js 15, Supabase, Google Sheets (escrita), Vitest.

**Spec:** `docs/superpowers/specs/2026-06-03-painel-embalagem-db-design.mdx`

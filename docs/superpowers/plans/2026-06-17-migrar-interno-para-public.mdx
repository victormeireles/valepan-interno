# Migrar schema `interno` → `public` — Plano de implementação

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Mover `whatsapp_notificacoes_config` para `public`, atualizar o app para ler `public`, e dropar o schema `interno` inteiro.

**Architecture:** Uma migration Supabase cria a tabela em `public`, copia a linha de config, e executa `DROP SCHEMA interno CASCADE`. O `WhatsAppConfigService` deixa de usar `.schema('interno')` e passa a usar o client padrão. Types regenerados via `npm run gen:types`.

**Tech Stack:** Supabase Postgres, Next.js 15, `@supabase/supabase-js`, Vitest.

**Spec:** `docs/superpowers/specs/2026-06-17-migrar-interno-para-public-design.mdx`

- [ ] **Step 1: Criar arquivo SQL**

```sql
-- DROP_INTERNO_MOVE_WHATSAPP_TO_PUBLIC.sql
-- Move whatsapp_notificacoes_config para public e remove schema interno abandonado.

CREATE TABLE IF NOT EXISTS public.whatsapp_notificacoes_config (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  embalagem_habilitado boolean NOT NULL DEFAULT false,
  fermentacao_habilitado boolean NOT NULL DEFAULT false,
  forno_habilitado boolean NOT NULL DEFAULT false,
  saidas_habilitado boolean NOT NULL DEFAULT false,
  updated_at timestamptz NOT NULL DEFAULT now()
);

ALTER TABLE public.whatsapp_notificacoes_config ENABLE ROW LEVEL SECURITY;

INSERT INTO public.whatsapp_notificacoes_config (
  id,
  embalagem_habilitado,
  fermentacao_habilitado,
  forno_habilitado,
  saidas_habilitado,
  updated_at
)
SELECT
  id,
  embalagem_habilitado,
  fermentacao_habilitado,
  forno_habilitado,
  saidas_habilitado,
  updated_at
FROM interno.whatsapp_notificacoes_config
WHERE NOT EXISTS (SELECT 1 FROM public.whatsapp_notificacoes_config LIMIT 1);

INSERT INTO public.whatsapp_notificacoes_config (
  embalagem_habilitado,
  fermentacao_habilitado,
  forno_habilitado,
  saidas_habilitado
)
SELECT false, false, false, false
WHERE NOT EXISTS (SELECT 1 FROM public.whatsapp_notificacoes_config LIMIT 1);

DROP SCHEMA IF EXISTS interno CASCADE;
```

## Ordem de execução crítica

**Migration (Task 1) deve rodar antes do deploy do código novo (Task 2).** Se deployar código novo sem migration, queries em `public` falham. Se rodar migration sem deploy, código antigo com `.schema('interno')` falha — janela mínima: aplicar migration e deploy imediato.

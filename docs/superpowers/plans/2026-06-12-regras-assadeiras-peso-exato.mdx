> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Refatorar `categoria_assadeira_regras` de faixas (`peso_min_g`/`peso_max_g`) para peso exato (`peso_g`), com match por igualdade no resolver e UI simplificada.

**Architecture:** `ALTER TABLE` na tabela vazia; domínio/actions/UI trocam min/max por `peso_g`; resolver usa `.eq('peso_g', pesoG)`; validação de overlap substituída por duplicata `(categoria_id, peso_g, assadeira_id)`. Cascata exceção → regra → pendente inalterada.

**Tech Stack:** Next.js 15, Supabase, Zod, Vitest, Tailwind, Material Icons.

```typescript
async function findDuplicateRegra(
  categoriaId: string,
  pesoG: number,
  assadeiraId: string,
  excludeId?: string,
): Promise<boolean> {
  const supabase = supabaseClientFactory.createServiceRoleClient();
  let query = supabase
    .from('categoria_assadeira_regras')
    .select('id')
    .eq('categoria_id', categoriaId)
    .eq('peso_g', pesoG)
    .eq('assadeira_id', assadeiraId)
    .eq('ativo', true);

  if (excludeId) query = query.neq('id', excludeId);

  const { data, error } = await query.maybeSingle();
  if (error && error.code !== 'PGRST116') {
    console.error('Erro ao verificar duplicata de regra:', error);
    return true;
  }
  return data != null;
}
```
